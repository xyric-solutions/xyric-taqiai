## Why this is one buildable unit
All category drafting pages share one engine (category → type → subType → verified template → Gemini generation), so the drafting core is a single buildable, verifiable unit: pick a category, supply facts, and confirm a structurally-compliant draft is produced without fabricated content.

## Acceptance (definition of done for the segment)
- [ ] Each category page exposes a `[type]/[subType]` selector and generates a draft only from a verified master template (no template → no draft).
- [ ] Generated drafts follow the template structure at 100% structural compliance; missing facts are flagged as gaps, never fabricated.
- [ ] Bilingual output: Urdu drafts render in Nastaliq with correct line-spacing; English drafts follow Pakistani legal formatting (`pk-format.ts`).
- [ ] Output HTML is sanitised via dompurify before render/storage.
- [ ] PDF export is available via the vault path (segment S1.5).
- [ ] Test expectation: integration tests assert template-gated generation, gap-flagging (no fabrication), and dompurify sanitisation of returned HTML.
- [ ] NFR checks: verified-template gate enforced; no fabricated facts; sanitised HTML; bilingual formatting correct.
- [ ] Verification: `POST /api/ai/generate` with a category+type+facts → HTML draft matching template sections with gaps flagged.
