## Acceptance (definition of done for the segment)
- [ ] A user can record in-browser or upload an audio file on `/voice-case` and receive a transcript.
- [ ] The transcript is verbatim — spoken content is transcribed as-is, not paraphrased, translated, or summarized.
- [ ] The original spoken language is preserved 100%: English stays English, Urdu stays Urdu script, Roman Urdu stays Roman, and code-switched/mixed speech keeps each segment in its spoken form.
- [ ] The transcript output is available for the next stage (feeds S3.3 case analysis → draft).
- [ ] Test expectation: integration test posts English-only, Urdu-script, Roman-Urdu, and mixed audio fixtures to `/api/ai/voice-transcribe` and asserts language/script preserved and no summarization (length/content parity with reference transcript).
- [ ] NFR checks: transcribe-only prompt (no translate/summarize); auth-scoped; audio handled within LLM gateway limits.
- [ ] Verification: `POST /api/ai/voice-transcribe` with a mixed-language recording → 200 returning a verbatim transcript in the original language(s).
