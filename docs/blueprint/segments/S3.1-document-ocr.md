## Why this is one buildable unit
Photo-to-text intake is a single vertical slice — one upload page plus one strict multi-step OCR API — that is verifiable end-to-end by uploading a document image and checking the returned text is a faithful, line-by-line transcription rather than a paraphrase.

## Acceptance (definition of done for the segment)
- [ ] A user can upload a photo/scan of a printed or handwritten legal document on `/copy-from-photo` and receive the extracted text.
- [ ] Extraction is strictly line by line: output preserves the source line order and returns exactly what is visible, never inferring/"correcting" to expected legal wording.
- [ ] Genuinely illegible portions are flagged as unclear rather than silently guessed (low false-confidence).
- [ ] The extracted text/HTML feeds downstream: image-typing in drafting and image translation (S3.4) consume the same extraction path.
- [ ] Test expectation: integration test posts a fixture image to `/api/ai/extract-document` and asserts returned line count + verbatim tokens (names/CNIC/section numbers) match ground truth; unclear-marker present when a region is masked.
- [ ] NFR checks: strict OCR (no invention); auth-scoped route; multi-step prompt runs within LLM gateway timeout.
- [ ] Verification: `POST /api/ai/extract-document` with a sample document image → 200 returning extracted text/HTML matching the source line for line.
