## Why this is one buildable unit
Persistence of drafts (list/create/get/update/soft-delete) plus PDF export is one CRUD surface over the `Document` model: buildable and verifiable in one pass by generating, saving, reloading, and exporting a document scoped to its owner.

## Acceptance (definition of done for the segment)
- [ ] A generated document can be saved with its `formData` (JSON) and `generatedContent` (HTML) and later retrieved by id.
- [ ] Documents are user-scoped: list/get/update/delete only return the authenticated owner's records.
- [ ] Update changes `status` (draft/published) and content; delete is a soft-delete (`deletedAt` set, row retained, excluded from listing).
- [ ] PDF export produces a downloadable PDF of the rendered document (html2pdf).
- [ ] Test expectation: integration tests cover create→list→get→update→soft-delete, owner-scoping enforcement, and that soft-deleted docs are excluded from list.
- [ ] NFR checks: ownership enforced on every record op; soft-delete preserves data; exported PDF matches rendered HTML.
- [ ] Verification: `POST /api/documents` then `GET /api/documents/[id]` → saved draft; `DELETE` → `deletedAt` set and item absent from `GET /api/documents`.
