## Why this is one buildable unit
It is a single review page plus one edit endpoint (HTML in → edited HTML out): buildable and verifiable in one pass by editing one clause of a draft and confirming every untouched section is returned byte-for-byte.

## Acceptance (definition of done for the segment)
- [ ] Given existing draft HTML and an edit instruction, the API applies only the requested change and returns the FULL updated HTML.
- [ ] Untouched content preservation is 100% — structure, formatting, and all unrelated sections are returned unchanged.
- [ ] `/draft-review` lets the lawyer see the current draft, request a change, and view the revised result.
- [ ] The same endpoint also backs translate-edit refinement (revising a prior translation result).
- [ ] Test expectation: integration tests assert single-clause edits leave all other nodes identical (diff limited to the targeted change).
- [ ] NFR checks: no unintended rewrites; returned HTML sanitised; full-document contract honoured.
- [ ] Verification: `POST /api/ai/edit-document` with draft HTML + one instruction → HTML where only the targeted section differs.
