## Why this is one buildable unit
This is the orchestration slice that wires an existing transcript (from S3.2) through case analysis (CAP-2) into drafting (CAP-1) — it reuses proven APIs and is verifiable end-to-end by running a recording all the way to a produced draft document.

## Acceptance (definition of done for the segment)
- [ ] From `/voice-case`, a produced transcript can be sent to case analysis and then to drafting without leaving the flow.
- [ ] Case analysis (`/api/ai/case-analysis`) decomposes the transcript into structured case components (parties, facts, issues, applicable law, suggested document).
- [ ] The analysis result flows into drafting (`/api/ai/case-prepare`, reused) to produce a drafted case document.
- [ ] End-to-end voice→draft completion is ≥ 70% (a typical advocate–client recording yields a usable draft without manual re-entry).
- [ ] Test expectation: integration test drives transcript → `/api/ai/case-analysis` → `/api/ai/case-prepare` and asserts a non-empty structured analysis plus a drafted document; E2E happy path from `/voice-case`.
- [ ] NFR checks: reuses grounded Phase-2 case analysis (C3); no fabricated facts introduced beyond the transcript; auth-scoped.
- [ ] Verification: `POST /api/ai/case-analysis` (transcript) → `POST /api/ai/case-prepare` (analysis) → 200 returning a drafted case document.
