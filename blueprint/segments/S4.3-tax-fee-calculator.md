## Why this is one buildable unit
One page reads one reference table (`stamp-duty-reference.ts`) and computes all charges arithmetically with no model call, so the calculator is a single buildable, verifiable unit: enter a value and province, get auditable, hand-reproducible numbers.

## Acceptance (definition of done for the segment)
- [ ] `/property-transfer/tax-calculator` computes Stamp Duty, CVT, PLRA, Registration Fee, FBR Withholding Tax (§236K buyer / §236C seller), and Capital Gains Tax per province (Punjab / Sindh / KPK / Balochistan / ICT).
- [ ] 100% deterministic: no LLM is ever called; identical inputs always yield identical numbers.
- [ ] Every line shows its rate and base so the result is reproducible by manual calculation.
- [ ] All rates/bases sourced from `src/lib/stamp-duty-reference.ts` (single source of truth).
- [ ] Test expectation: UNIT tests on rate/base correctness per province and edge inputs (zero/large values, filer vs non-filer where applicable).
- [ ] NFR checks: determinism (no generative path); auditability (rate + base displayed for every line).
- [ ] Verification: enter a known property value + province → each charge equals the manual `base × rate`, matching the reference table.
