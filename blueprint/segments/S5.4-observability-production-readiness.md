## Why this is one buildable unit
Error tracking, structured logs, a healthcheck, and a deploy checklist are the single "can we see and trust production" capability; they wire into the same logging/config layer and are verified together by observing a real deploy.

## Acceptance (definition of done for the segment)
- [ ] Error tracker / APM (Sentry or equivalent) captures unhandled server + client errors in production.
- [ ] Structured logs emitted for AI calls, auth failures, rate-limit hits, and search failures (queryable fields, no PII/secrets).
- [ ] Healthcheck endpoint returns green when app + DB reachable, red otherwise.
- [ ] Deployment verification checklist for Railway exists and is runnable step-by-step.
- [ ] Test expectation: unit test that the logger emits structured JSON with required fields; integration test hitting the healthcheck (200 healthy / 503 degraded); smoke test that a thrown error reaches the tracker sink.
- [ ] NFR checks: logs contain no secrets or PII; healthcheck cheap (no heavy queries); tracker sampling configured to control cost.
- [ ] Verification: `GET /api/health` → 200 with `{status:"ok", db:"up"}`; trigger a test error → appears in the tracker dashboard.
