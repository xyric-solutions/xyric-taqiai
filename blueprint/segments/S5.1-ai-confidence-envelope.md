## Why this is one buildable unit
Every AI feature returns free-form JSON with no trust signal today (audit P0-01); defining ONE envelope and threading it through all AI endpoints + Prisma + UI is a single cross-cutting contract verifiable end-to-end in one pass.

## Acceptance (definition of done for the segment)
- [ ] Confidence envelope present on 100% of AI endpoint responses (`/api/ai/*`).
- [ ] Low-confidence (band `low`) outputs are visually and distinctly flagged in the UI adjacent to the AI content.
- [ ] Prisma `Document` and `ChatMessage` persist composite confidence + band + model_version.
- [ ] Test expectation: unit tests for the envelope builder/validator; integration tests asserting every AI route returns a well-formed envelope; E2E asserting a low-confidence banner renders. Coverage ≥ 80% on envelope lib.
- [ ] NFR checks: envelope schema is backward-compatible (data still readable by existing consumers); no confidence value ever fabricated by the model — bands derived from computed dimensions.
- [ ] Verification: `POST /api/ai/generate` (sample facts) → response includes `confidence.composite`, `confidence.band`, and all four `confidence.dimensions`.
