Legal translation is one page backed by three cooperating endpoints (free, template-based, refine) sharing one preservation contract — it is verifiable end-to-end by translating a document and confirming every proper noun, number, and date is carried over exactly.

## Acceptance (definition of done for the segment)
- [ ] A user can translate legal text or a document image between Urdu, English, and Arabic on `/translate`.
- [ ] Free translation preserves legal meaning and document structure; every proper noun, number, and date is carried over 100% unchanged.
- [ ] Structured template-based translation works for known Pakistani documents (Fard, Nikah Nama, ID Card, deeds, certificates) via `/api/ai/translate-template`.
- [ ] A prior translation can be refined/corrected via `/api/ai/translate-edit` without corrupting untouched parts.
- [ ] Image translation reuses the strict OCR path (S3.1) so scanned/photographed documents translate from faithful extraction.
- [ ] Test expectation: integration tests assert proper-noun/number/date parity between source and translation (free + template modes) and that translate-edit changes only the requested portion.
- [ ] NFR checks: 100% entity preservation (names/numbers/dates); structure preserved; auth-scoped routes.
- [ ] Verification: `POST /api/ai/translate` with legal text containing names, CNIC, amounts, and dates → 200 with all such tokens intact in the translated output.
